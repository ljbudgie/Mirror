# Mirror — User Guide

Mirror helps you understand your rights and take one clear next step when an
institution has wronged you. It works entirely on your device. Nothing you type
is ever sent to a server.

---

## Getting started

### Requirements

- Python 3.11 or later

### Install Python dependencies

```bash
pip install -e ".[dev]"
```

### Run the web interface

Open `web/index.html` in your browser. No server needed.

### Use the Python toolkit directly

```bash
# Classify your situation
python -c "from core.conversation import classify; print(classify('My landlord is ignoring my repair requests'))"

# Get your applicable rights
python -c "
from core.conversation import classify
from core.rights_mapper import map_rights
domain = classify('The DWP has refused my PIP claim')
rights = map_rights(domain)
for r in rights.rights:
    print(f'- {r.name} ({r.source})')
"

# Get your next step
python -c "
from core.next_step import get_next_step, Stage
step = get_next_step('benefits', Stage.INITIAL)
print(step.action)
print()
print(step.detail)
"

# Render a template
python -c "
from core.template_engine import render
print(render('sar_request', {
    'your_name': 'Jane Smith',
    'your_address': '1 Example Street, London, EC1A 1BB',
    'your_email': 'jane@example.com',
    'today_date': '11 April 2026',
    'organisation_name': 'Example Organisation',
    'organisation_address': '2 Corp Lane, London, EC2A 2BB',
    'your_full_name': 'Jane Smith',
    'your_dob': '01 January 1990',
    'reference_number': 'N/A',
}))
"
```

## The web interface

Open `web/index.html` in any modern browser. Type what happened in plain
English. Mirror will:

1. Work out what kind of situation you're in
2. Show you the rights that apply
3. Give you exactly one next step

You can say **draft** at any point and Mirror will tell you how to generate
the letter or email locally.

## Domains Mirror understands

| Domain | Examples |
|---|---|
| Enforcement | Police, fines, regulatory action |

| Housing | Landlords, eviction, repairs, deposits |

| Immigration | Visa refusals, Home Office decisions, appeals |

## Configuration

Edit `mirror-config.json` to set your preferences:

```json
{
  "jurisdiction": "UK",
  "language": "en",
  "accessibility": {
    "large_text": false,
    "screen_reader": false,
    "email_only": false
  },
  "export_format": "txt",
  "vault_path": null,
  "templates_dir": null,
  "ai": {
    "enabled": false,
    "model": "mistral",
    "base_url": "http://localhost:11434",
    "timeout": 120
  }
}
```

**Jurisdiction** options: `"UK"` (default) or any ISO country code. Non-UK
jurisdictions receive international framework guidance.

**Accessibility**: set `large_text` or `screen_reader` to `true` and Mirror
adjusts its output format accordingly.

**Export format**: `"txt"` (default) or `"pdf"` (requires additional setup).

**Vault path**: set `vault_path` only if you want commitment salts stored in a
specific local encrypted vault. Leave it as `null` to use Mirror's default.

**Templates directory**: set `templates_dir` only if you want to use a local
custom template folder.

**AI**: optional local AI is disabled by default. If enabled, Mirror expects a
local backend such as Ollama at `base_url` and falls back to deterministic
classification if the backend is unavailable.

## The Burgess Principle

Every outgoing communication generated by Mirror includes the Burgess
Principle question. This question asks the organisation to confirm:

- Whether any automated decision-making was involved
- The logic and criteria applied
- A complete and accurate account of the relevant decisions

The Burgess Principle is a registered accountability methodology
(UK00004343685). It is designed to surface information that organisations
are legally required to provide but often do not volunteer.

## Commitment hashing

Mirror generates a unique SHA-256 commitment hash for every communication you
send. This lets you prove — without revealing personal data — that a specific
message originated from you at a specific time.

Salts are stored in an encrypted local vault at `~/.mirror/vault.enc`.
Your vault passphrase never leaves your device.

Use a commitment hash when you send an important request, complaint, appeal, or
trusted-contact authority. Keep the hash, timestamp, and local vault entry with
your records so you can later verify what was sent without publishing the
message itself.

## Timeline tracking

Mirror tracks statutory deadlines:

| Process | Deadline |
|---|---|
| Subject Access Request (DSAR) | 30 calendar days |
| Freedom of Information (FOI) | 20 working days |
| NHS complaint acknowledgement | 3 working days |
| Mandatory Reconsideration | 1 calendar month |
| Employment Tribunal claim | 3 months minus 1 day |

When a deadline is approaching or missed, Mirror generates the appropriate
follow-up or escalation letter automatically.

For local evidence discipline, keep a simple export bundle for each matter:
your situation summary, rights map, next step, chosen template, deadline, and
commitment hash metadata. This can remain as a local text file; no account or
server is required.

## Privacy

Mirror is local-first by design.

- No data leaves your device
- No accounts or registration
- No analytics or tracking
- No cookies
- The web interface has no network requests
- Your vault is encrypted on your device
